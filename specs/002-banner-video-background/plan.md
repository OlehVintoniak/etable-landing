# Implementation Plan: Banner Video Background

**Branch**: `002-banner-video-background` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/002-banner-video-background/spec.md`

## Summary

Modify the existing `BannerComponent` to render a full-bleed looped video behind the company name, tagline, and CTA button. The video is supplied via an optional `videoSrc` input; when absent the component is visually unchanged. A semi-transparent dark overlay maintains WCAG AA text contrast across all video frames. The `<video>` element uses HTML attributes only (`autoplay muted loop playsinline`) — no JavaScript playback API is called — making the change SSR-safe with no additional guards.

## Technical Context

**Language/Version**: TypeScript 5.x / Angular 21.2 (standalone components, OnPush)  
**Primary Dependencies**: `@angular/common` (isPlatformBrowser — already in use), `@angular/ssr` (baseline)  
**Storage**: N/A  
**Testing**: PROHIBITED by constitution (manual validation only)  
**Target Platform**: Browser (Chrome, Firefox, Safari, Edge — desktop + mobile) + Node.js SSR  
**Project Type**: Web application (Angular SSR/SSG)  
**Performance Goals**: No layout shift on hydration; video plays within standard browser autoplay policy  
**Constraints**: SSR-safe (no JS errors on server), WCAG AA 4.5:1 contrast, `object-fit: cover` fill, no new npm dependencies  
**Scale/Scope**: Single component modification (`BannerComponent`) + one template line change (`home.html`)
